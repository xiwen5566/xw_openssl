size_t CRYPTO_128_unwrap ( void *  key,

const unsigned char *  in,
size_t  inlen,
block128_f  block 
)

Unwrapping according to RFC 3394 section 2.2.2, including the IV check. The first block of plaintext has to match the supplied IV, otherwise an error is returned.

Parameters
[in]keyKey value.
[out]ivIV value to match against. Length = 8 bytes. NULL = use default_iv.
[out]outPlaintext without IV. Minimal buffer length = (inlen - 8) bytes. Input and output buffers can overlap if block function supports that.
[in]inCiphertext as n 64-bit blocks.
[in]inlenLength of in.
[in]blockBlock processing function.
Returns
0 if inlen is out of range [24, CRYPTO128_WRAP_MAX] or if inlen is not a multiple of 8 or if IV doesn't match expected value. Output length otherwise.
